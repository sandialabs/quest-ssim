Installation¶

The SSim package is available on pypi. You can install it via:

pip install quest-ssim

Running the simulation¶

You may launch the GUI with the ssim command, or use HELICS CLI to run the simulator directly. An example configuration for the HELICS CLI is provided in examples/demo_670/federation.json (shown below):

{
  "federates": [
    {
      "directory": ".",
      "exec": "helics_broker -f 4",
      "host": "localhost",
      "name": "broker"
    },
    {
      "directory": ".",
      "exec": "storage-federate S1 --hours 1025 grid.json ../../ssim/federates/storage.json",
      "host": "localhost",
      "name": "s1"
    },
    {
      "directory": ".",
      "exec": "grid-federate --hours 1025 grid.json ../../ssim/federates/grid.json",
      "host": "localhost",
      "name": "grid"
    },
    {
      "directory": ".",
      "exec": "reliability-federate --hours 1025 grid.json ../../ssim/federates/reliability.json",
      "host": "localhost",
      "name": "reliability"
    },
    {
      "directory": ".",
      "exec": "logger-federate --hours 1025 grid.json ../../ssim/federates/logger.json",
      "host": "localhost",
      "name": "logger"
    }
  ],
  "name": "ssim_demo"
}

Running this with:

helics run --path examples/demo_670/federation.json

will start the grid federate, two storage controllers, the reliability federate, and the logger federate. When the simulation finishes a variety of matplotlib figures will be displayed.

For more information on configuring each federate see Simulation Configuration.

Each federate has its own console_scripts entry point, which take several command line arguments specifying how long to run for and the paths to the configuration files. These entry points are:

  • storage-federate

  • grid-federate

  • reliability-federate

  • logger-federate

For an overview of the options you can pass to each entry point use --help. For example:

storage-federate --help
